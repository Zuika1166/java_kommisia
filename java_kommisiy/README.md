# Service Center Warehouse

Консольное приложение на Java 17+ для учета запчастей на складе сервисного центра и регистрации заказов на ремонт.

## Технологии

- Java 17+
- JDBC
- SQLite
- Maven

## Возможности

- CRUD для запчастей
- CRUD для заказов
- поиск запчастей по названию через `LIKE`
- фильтрация заказов по статусу
- транзакционное оформление заказа
- конфигурация БД через `.properties`

## Архитектура

- `model` - доменные модели `Part`, `Order`, `OrderStatus`
- `dao` - интерфейсы доступа к данным
- `dao.impl` - JDBC-реализации DAO
- `service` - бизнес-логика
- `ui` - консольное меню
- `config` - подключение и инициализация БД

## База данных

При первом запуске приложение автоматически:

- создает файл `service-center.db`
- создает таблицы из `schema.sql`
- добавляет тестовые данные из `data.sql`

## Ключевой сценарий

При оформлении заказа приложение:

1. проверяет наличие запчасти на складе;
2. уменьшает остаток;
3. создает запись о заказе;
4. фиксирует все через одну транзакцию.

Если возникает ошибка, выполняется `rollback()`.
